# 小红书帖文内容分析与概括指南

## 任务概述
当用户提供一条或多条小红书链接时，需要依次使用 chrome-devtools 的 MCP 工具访问这些链接，全面分析每条帖文的内容与评论，并合并生成覆盖所有帖文的标准化概括文本。指南适用于单帖和多帖并行场景。

## 操作步骤

### 1. 收集与分组链接
- 使用 `chrome-devtools__navigate_page` 依次打开用户提供的每个链接，确认页面加载成功。
- 如多个帖文围绕同一事件，需归为一个主题组；主题不同则分别输出概括。
- 记录帖文与链接的对应关系，避免遗漏或混淆。

### 2. 分析帖文主体信息
- 为每条帖文记录发帖时间（换算为具体日期）、发帖用户昵称（使用 @ 标注）、核心内容与诉求、附件素材要点（如图片或视频透露的事实）。
- 标注帖文的互动数据：点赞数与评论数（收藏数可忽略）。
- 若多条帖文相互补充事实，需整合形成完整事件链；若信息存在冲突，应在概括中说明差异。

### 3. 收集评论观点
- 结合 `chrome-devtools__evaluate_script` 执行滚动操作（如 `window.scrollBy(0, window.innerHeight)`），必要时循环多次以加载更多评论。
- 使用 `chrome-devtools__take_snapshot` 获取包含评论区的页面结构，可按需要多次截取以覆盖全部可见评论。
- 汇总所有帖文中的主要评论观点，按观点归类，避免重复。
- 标注评论观点对应的帖文范围（是否为多帖共性观点或仅针对某条帖文）。

### 4. 整理互动数据
- 如果是单条帖文，直接给出单帖互动量
- 如果是多条帖文，给出总互动量（多帖相加）

### 5. 撰写输出
- 按下方格式模板撰写结果，并以 `★` 开头拟定标题概括主题。
- 主体段落先说明发帖时间范围与主要账号，再综合叙述事件事实。
- 评论概括使用“部分网民表示/认为/质疑/建议……”句式，不同观点用分号分隔。
- 在结尾括号中列出所有链接，可使用顿号分隔。

## 信息提取要点
- **发帖时间**：将相对时间转换为具体日期（例：“3 天前”需换算成“9 月 2 日”）。
- **用户昵称**：按帖文显示名称记录，使用 `@昵称` 形式。
- **核心事实**：概括每条帖文中的关键事实、诉求或最新进展。
- **互动数据**：至少记录点赞和评论数量，可给出合计与分帖数据。
- **评论观点**：归纳主要观点，区分不同类别。
- **差异说明**：多帖存在不同结论或更新时，需在概括中强调。
- **院校/机构名称**：标题和正文提及学校或机构时需写全称（例：“北航”写作“北京航空航天大学”，“中传”写作“中国传媒大学”）。

## 输出格式规范

```
★网民发帖[用简洁动词概括争议点]

[时间概括]，小红书平台用户“@[用户A]”“@[用户B]”等发布多条帖文，称[主题核心概括]。截至目前，上述帖文合计获[总点赞数]次点赞、[总评论数]条评论。部分网民表示[观点1]；部分网民表示[观点2]；部分网民表示[观点3]；部分网民表示[观点4]。（小红书[链接1]；[链接2]；[链接3]）
```

### 使用说明
- 若只有单条帖文，可省略“多条”表述及分帖数据，按单帖格式输出。
- 标题通常以“网民发帖”开头，紧跟动词与具体议题，例如：“网民发帖吐槽北京学生补课压力大”“网民发帖反映北京外国语大学食堂餐食藏虫引发学生担忧”“网民发帖称北京大学食堂菜价走高补贴难见成效”；不得改写或省略该固定前缀。
- 标题需客观中性，突出争议或问题点，避免主观评价。
- 事实段落按照“时间—用户—事件—数据”逻辑展开，确保信息完整。
- 评论段落建议保留 3-4 条核心观点，可跨帖合并，保持条理。
- 避免使用“网友”称呼，称呼统一为“网民”。
- 链接按用户提供的顺序列出，App 内短链可直接引用。

## 固化流程建议

### 典型输入格式
```
[标题描述] [小红书链接] 复制后打开【小红书】查看笔记！
```

### 处理流程
1. 识别并整理全部小红书链接，按主题分组。
2. 针对每个主题组，依照本指南完成访问、信息提取与多帖整合。
3. 输出与主题组数量等同的概括段落，确保所有链接都在最终文本中引用。

### 自动化识别
- 系统会自动提取以 `http://xhslink.com/` 或 `https://www.xiaohongshu.com/` 开头的链接。
- “复制后打开【小红书】查看笔记！”等提示语可忽略。
- 当用户一次提供多条文本时，保持原有顺序处理并在最终结果中全部覆盖。
