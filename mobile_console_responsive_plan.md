# 人工筛选控制台移动端适配计划

## 项目现状与痛点
- 前端基于 FastAPI + Jinja + 原生 JS，唯一样式文件 `src/console/web/static/css/dashboard.css`；主要页面模板 `src/console/web/templates/manual_filter.html`。
- 布局为桌面优先：容器固定最大宽度 1000px，`header`、`tabs`、`filter-layout`、`review-grid` 等多列 flex/grid 在窄屏下无断点；按钮/表单尺寸固定，radio 组横向展开易换行溢出。
- 交互由 `src/console/web/static/js/dashboard.js` 驱动，逻辑与布局耦合较少，主要是基于 class/ID 选择器渲染卡片，改样式即可。

## 目标
- 让 `/manual_filter` 在手机端（<= 768px）可用、可读：避免横向滚动，重要信息优先展示，操作按钮可触达。
- 保持桌面端现有布局/样式体验不变或轻微调整。

## 设计思路
- 按断点（建议 1024px、768px）增加响应式样式，优先使用 CSS（不改 JS 逻辑）。
- 采用“单列 + 可折叠”理念：过滤页侧边栏改为顶部横向滚动按钮；文章卡片收紧间距，元信息栈式排列；批量操作条与工具栏堆叠。
- 审阅页网格在小屏改为单列；导出弹窗改为全屏/大屏占比，按钮全宽。

## 主要改动项
1) **基础布局**
   - `body`/`container` 改为流式宽度，左右内边距基于 viewport；header 支持换行，stats 区域可滚动或换行。
   - Tabs 与 toolbar 调整为自适应宽度，必要时纵向堆叠。
2) **筛选页（Filter）**
   - `filter-layout` 在窄屏切为单列；侧边按钮改为横向滚动条或两列网格。
   - 文章卡片 padding/字体缩放，meta 行改为两列/多行包裹；radio 组允许换行并保持触摸尺寸。
   - `actions-bar` 在移动端改为静态/底部全宽按钮，避免遮挡内容。
3) **审阅页（Review）**
   - `review-grid` 在 <1024px 切为单列；排序模式下 drag handle 与状态选择保持一行。
   - 批量工具栏允许换行，按钮/下拉全宽。
4) **放弃与弹窗**
   - Discard 列表卡片收紧；分页按钮全宽。
   - 导出 Modal 在移动端占满宽度与大部分高度，内部滚动，按钮纵向排布或双列。
5) **可用性细节**
   - 增大按钮/输入控件的触摸区域（最小 40px 高度）。
   - 调整字体级别（标题、正文、次要信息），保证对比度。

## 交付与验证
- 交付：更新 `dashboard.css` 响应式样式，必要时对 `manual_filter.html` 做轻量结构调整（如包裹容器/类名），不修改核心 JS 逻辑。
- 验证：桌面宽度 1280px 验证无回归；移动端 430px、768px 下逐 tab 自测（筛选/审阅/放弃/导出），确认滚动、批量操作、拖拽排序（如适用）可用。
- 如时间允许，补充 README 段落记录移动端使用提示与已知限制。
