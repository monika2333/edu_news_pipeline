中国教育报详情页副标题拼接方案
================================

## 目标
- 详情页如存在副标题，将其合并到最终标题中，保留原有符号与顺序。
- 兼容副标题位于主标题前后两种模板。
- 不调整列表页抓取逻辑，仅在详情页补齐标题。

## 现状
- 列表页标题来自搜索 API `title` 或列表页链接文本。
- 详情页标题只取 `<title>` 或 `h1`，随后覆盖列表页标题。
- 示例页面结构：
  - `...5874.html`：`h1` 后紧跟 `h4`，内容为“——2026年全国教育工作会议观察之三”。
  - `...5889.html`：`h3` 在 `h1` 之前，内容为“安庆师范大学智慧农业团队技术赋能乡村振兴——”。
  - `...5891.html`：`h3` 在 `h1` 之前，内容为“南京航空航天大学：”。
  - 同一父容器中还会有 `h2`，但该节点为发布时间/作者/来源等元数据。

## 方案
- 在详情页解析中新增标题抽取逻辑：
  - 以 `h1` 为主标题。
  - 在 `h1` 的父节点内分别查找 `h3`（前置副标题）与 `h4`（后置副标题），二者独立取值；不存在则跳过；`h2` 明确忽略（它是发布时间/作者/来源等元数据，不属于标题）。
  - 组合为：`前置副标题 + 主标题 + 后置副标题`。
- 组合后统一为单行输出；不新增符号、不插入空格。
- 若未找到副标题，一切维持原状（`h1` 覆盖 `<title>` 的现有逻辑保持不变）。

## 实施步骤
1. 在 `fetch_detail()` 内增加一个私有辅助函数，抽取 `h1`、`h3`、`h4` 文本。
2. 组合出最终标题并回填到返回的 `title` 字段。
3. 用三条样例 URL 做人工验证，确认标题输出符合预期：
   - `...5874.html` 期望：`以砥砺实干笃行“十五五”新征程——2026年全国教育工作会议观察之三`
   - `...5889.html` 期望：`安庆师范大学智慧农业团队技术赋能乡村振兴——书写田埂间的“科技答卷”`
   - `...5891.html` 期望：`南京航空航天大学：教师党支部书记比拼素质能力`

## 已确认
- 标题统一为单行输出。
- 目前仅覆盖已提供的页面模板（`h3` 前置副标题 + `h1` 主标题，或 `h1` + `h4` 后置副标题）。
- 不需要处理同时存在 `h3` 与 `h4`、多段标题等情况。
- 页面结构稳定，不需要考虑容器深度或跨父容器的查找。
